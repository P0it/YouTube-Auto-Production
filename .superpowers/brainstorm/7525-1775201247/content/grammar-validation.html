문장 검증 방식 비교

Claude API 과금 vs 대안 — 비용 효율적인 문법 검증 방법

핵심 관점: 이 게임의 문장은 유한한 카드 세트에서 조합됩니다. GPT/Claude 같은 범용 LLM이 아니라, 더 가볍고 저렴한 방법으로 충분할 수 있습니다.

비용 시뮬레이션 (월 10만 게임 기준)

방식 호출당 비용 월 예상 비용
Claude API (Haiku) ~$0.001 ~$100+
자체 호스팅 소형 모델 ~$0.0001 ~$20-50 (서버비)
규칙 기반 + NLP 라이브러리 $0 $0 (서버 연산만)
하이브리드 (추천) 대부분 $0 ~$5-10
A

규칙 기반 + 패턴 매칭 (추천)

카드가 유한하다는 점을 활용 — LLM 없이 해결

원리:
카드 세트가 정해져 있으므로, 가능한 문장 패턴을 사전 정의할 수 있습니다.

카드: [I] [like] [cats] [the] [big]
유효: SVO → "I like cats" ✅
유효: SVO+Adj → "I like the big cats" ✅
무효: "cats like I" ❌ (어순 오류)
구현 방식:
1. 각 카드에 품사 태그 (명사, 동사, 형용사 등)
2. 문장 구조 템플릿 정의 (SVO, SVC, SVOO 등)
3. 플레이어 배열이 템플릿에 매칭되는지 검사
4. 난이도별로 허용 템플릿 확장

장점

  • 비용 $0 — API 호출 없음
  • 즉시 판정 (네트워크 지연 없음)
  • 오프라인에서도 동작
  • 결정론적 — 같은 입력 항상 같은 결과
  • 초급 레벨에 완벽히 적합

단점

  • 복잡한 문장(관계절 등)은 패턴이 폭발적으로 증가
  • "문법적으로 맞지만 의미가 이상한 문장" 판별 불가
B

하이브리드: 규칙 기반 + 소형 모델 (백업)

90%는 규칙으로, 10% 애매한 경우만 모델 호출

2단계 검증:

1단계 (로컬/서버): 품사 기반 규칙 검사
→ 확실한 정답/오답은 여기서 즉시 판정

2단계 (소형 모델): 애매한 경우만
→ HuggingFace의 grammar-checker 모델 (서버 호스팅)
→ 또는 Cloud Functions에서 경량 NLP 처리
소형 모델 옵션:
• T5-small (grammar correction) — 242MB, CPU로 충분
• LanguageTool API — 오픈소스 문법 검사, 무료 셀프호스팅
• spaCy dependency parser — 문장 구조 분석, 매우 가벼움

장점

  • 비용 극소 (대부분 규칙으로 처리)
  • 복잡한 문장도 커버 가능
  • 고급 레벨로 확장할 때 유연함

단점

  • 소형 모델 서버 호스팅 필요 (월 $20-50)
  • 2단계 호출 시 약간의 지연
C

로컬 LLM (온디바이스)

HuggingFace 모델을 디바이스에서 직접 실행

⚠️ 현실적 제약:
• 모바일에서 LLM 실행은 배터리 소모 극심
• 최소 500MB~2GB 모델 다운로드 필요
• 추론 시간 1~5초 (게임 흐름 방해)
• 저사양 기기 지원 불가
• 아이들 기기는 대부분 저사양

장점

  • 서버 비용 $0
  • 오프라인 동작
  • 프라이버시

단점

  • 모바일 디바이스에서 LLM은 비현실적
  • 앱 용량 폭증 (1GB+)
  • 배터리/발열 문제
  • 타겟이 아이들 — 저사양 기기 많음

💡 추천: A(규칙 기반) 시작 → B(하이브리드)로 확장

카드 세트가 유한하다는 것이 핵심입니다. 초급~중급 레벨의 문장은 품사 태그 + 문장 구조 템플릿으로 100% 판정 가능합니다. 고급 레벨에서 복잡한 문장이 등장할 때만 서버의 소형 모델(T5-small 또는 LanguageTool)을 사용하면 비용을 월 $5~10 수준으로 유지할 수 있습니다.

Claude API는 AI 대전 플레이어 로직에만 사용하면 과금을 최소화할 수 있습니다.
