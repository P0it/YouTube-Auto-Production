최종 기술 스택 비교 (Rive 기반)

A

Flutter + Flame + Rive (추천)

게임 로직은 Flame, 캐릭터는 Rive, UI는 Flutter

기술 구성:
• 게임 보드/카드: Flame Engine (2D 게임 렌더링)
• 캐릭터 애니메이션: Rive (듀오링고와 동일)
• UI/메뉴: Flutter 위젯
• 실시간 대전: Firebase Realtime + Cloud Functions
• AI 대전: Claude API (문장 검증 + AI 플레이어)
• 빌드: iOS + Android + Web

장점

  • Flame(게임) + Rive(캐릭터) + Flutter(UI) 조합이 이 게임에 딱 맞음
  • 듀오링고와 동일한 애니메이션 스택
  • 하나의 코드베이스로 iOS + Android + Web
  • 카드 드래그/부채꼴 배열을 Flame이 자연스럽게 처리
  • Unity보다 훨씬 가벼운 빌드 (10~20MB)

단점

  • Dart 학습 필요 (1~2주면 익숙해짐)
  • Flutter Web은 모바일 대비 약간 무거움
B

React Native + Skia + Rive

카드는 Skia 커스텀 렌더링, 캐릭터는 Rive

기술 구성:
• 게임 보드/카드: React Native Skia + Reanimated
• 캐릭터 애니메이션: Rive (rive-react-native)
• UI/메뉴: React Native 컴포넌트
• 실시간 대전: Supabase Realtime (WebSocket)
• AI 대전: Claude API
• 빌드: iOS + Android (+ React Native Web)

장점

  • JavaScript/TypeScript — 학습 부담 적음
  • Rive로 캐릭터 애니메이션 동일하게 가능
  • gstack 워크플로우 호환
  • Expo 생태계의 빠른 개발 사이클

단점

  • Skia로 카드 게임 보드를 직접 구현해야 함 (Flame같은 게임 엔진 없음)
  • 부채꼴 카드 배열, 드래그 등을 수동 구현
  • React Native Web은 게임 UI에서 제한적

🎯 추천: A — Flutter + Flame + Rive

이유: 듀오링고급 캐릭터 애니메이션(Rive) + 카드 게임 전용 엔진(Flame) + 크로스플랫폼(Flutter)의 조합이 이 프로젝트에 가장 적합합니다.

React Native도 Rive 캐릭터는 동일하게 구현 가능하지만, 카드 게임 보드 자체(부채꼴 배열, 드래그앤드롭, 카드 뒤집기 등)를 만들 게임 엔진이 없어 직접 구현해야 합니다. Flame은 이런 2D 게임 로직을 자연스럽게 처리합니다.
