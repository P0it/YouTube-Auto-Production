검증 규칙 예시

규칙 1: 관사 (a / an / the)
a + cat ✅
다음 단어 vowelStart=false → "a"
an + apple ✅
다음 단어 vowelStart=true → "an"
a + apple ❌
vowelStart=true인데 "a" 사용
규칙 2: 주어-동사 수일치
He likes cats ✅
3인칭 단수 + likes(3sg)
They like cats ✅
3인칭 복수 + like(base)
He like cats ❌
3인칭 단수인데 base form
규칙 3: 형용사 어순
big red ball ✅
크기(1) → 색상(6) 순서 OK
red big ball ⚠️
색상(6) → 크기(1) 어색하지만
초급에선 허용할 수도

영어 형용사 어순: 의견 → 크기 → 나이 → 모양 → 색상 → 출신 → 재질 → 목적
각 형용사 카드에 adjOrder: 1~8 태깅

규칙 4: 단수/복수 일치
a cat ✅
부정관사 + 단수 OK
a cats ❌
부정관사 + 복수 불가
the cats ✅
정관사 + 복수 OK

규칙으로 커버가 어려운 경우 (고급 레벨)

의미적 타당성

"The chair eats a book" — 문법은 맞지만 의미 이상
→ 해결: 카드 세트를 의미적으로 호환되는 그룹으로 구성

복잡한 절 구조

"The cat that I saw yesterday..." — 관계절
→ 해결: 고급 레벨에서만 등장, 이때 소형 모델 사용

결론: 규칙 기반이면 충분합니다

카드에 품사, 인칭, 수, 관사호환성, 형용사순서 등의 메타데이터를 태깅하면, 관사/단복수/형용사 어순/주어동사 수일치 등 대부분의 문법 규칙을 LLM 없이 처리할 수 있습니다.

의미적으로 이상한 문장은 카드 세트 설계로 방지하고, 관계절 같은 복잡한 구조는 고급 레벨에서만 소형 모델로 보완하면 됩니다.
