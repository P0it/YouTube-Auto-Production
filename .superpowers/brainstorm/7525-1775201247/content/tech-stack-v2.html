A

🔥 Unity + WebGL (추천 변경)

게임 엔진: Unity (C#)

애니메이션: Spine / Unity Animator (캐릭터 뼈대 애니메이션)

멀티플레이: Unity Netcode + Supabase (매칭/인증)

빌드: iOS, Android 네이티브 + WebGL (브라우저)

광고: Unity Ads + AdMob Mediation

캐릭터 애니메이션 ⭐⭐⭐⭐⭐ Spine + Animator 완벽 지원
카드 인터랙션 ⭐⭐⭐⭐⭐ 물리엔진, 파티클 네이티브
이모트 시스템 ⭐⭐⭐⭐⭐ 이펙트 + 사운드 통합 쉬움
실시간 대전 ⭐⭐⭐⭐ Netcode + WebSocket
웹 지원 ⭐⭐⭐ WebGL 빌드 가능, 로딩 느림
개발 난이도 ⭐⭐⭐ C# 필요, 러닝커브 있음

장점

  • 카드 게임의 모든 요소를 자연스럽게 구현 가능
  • Spine으로 듀오링고급 캐릭터 애니메이션
  • 파티클, 이펙트, 사운드 통합이 게임 엔진 답게 강력
  • 모바일 + 웹(WebGL) 모두 빌드 가능
  • 카드 게임 레퍼런스/에셋 풍부

단점

  • C# 학습 필요
  • WebGL 빌드 초기 로딩이 느릴 수 있음 (5~15초)
  • gstack 워크플로우 직접 호환은 안됨
  • 앱 용량이 커질 수 있음 (50MB+)
B

Flutter + Flame Engine

게임 엔진: Flame (Flutter 네이티브 2D 게임 엔진)

애니메이션: Rive (인터랙티브 캐릭터 애니메이션)

멀티플레이: Firebase Realtime DB + Cloud Functions

빌드: iOS, Android, Web (Flutter 통합)

광고: Google AdMob (Flutter 플러그인)

캐릭터 애니메이션 ⭐⭐⭐⭐ Rive로 인터랙티브 애니메이션
카드 인터랙션 ⭐⭐⭐⭐ Flame + 제스처 잘 지원
이모트 시스템 ⭐⭐⭐⭐ Rive 스테이트 머신으로 구현
실시간 대전 ⭐⭐⭐⭐ Firebase Realtime 안정적
웹 지원 ⭐⭐⭐ Flutter Web은 아직 무거움
개발 난이도 ⭐⭐⭐⭐ Dart 배우기 비교적 쉬움

장점

  • 하나의 코드베이스로 모바일 + 웹
  • Rive로 듀오링고급 인터랙티브 캐릭터 가능
  • Flame이 2D 게임에 최적화
  • Unity보다 가볍고 빠른 빌드

단점

  • Dart 학습 필요
  • Flame 생태계가 Unity보다 작음
  • 복잡한 이펙트는 직접 구현 필요
  • Flutter Web 성능 아직 제한적
C

React Native + Skia + Lottie

프레임워크: React Native (Expo)

애니메이션: React Native Skia + Lottie (After Effects 애니메이션)

멀티플레이: Supabase Realtime (WebSocket)

빌드: iOS, Android (네이티브) + React Native Web

광고: Google AdMob

캐릭터 애니메이션 ⭐⭐⭐ Lottie로 가능하나 인터랙티브 제한
카드 인터랙션 ⭐⭐⭐⭐ Reanimated + Gesture Handler
이모트 시스템 ⭐⭐⭐ Lottie 애니메이션 재생은 가능
실시간 대전 ⭐⭐⭐⭐ Supabase Realtime 좋음
웹 지원 ⭐⭐⭐ React Native Web 제한적
개발 난이도 ⭐⭐⭐⭐⭐ JS/TS 생태계, 가장 쉬움

장점

  • JavaScript/TypeScript — 가장 익숙한 생태계
  • Skia로 커스텀 2D 렌더링 가능
  • gstack 워크플로우 완벽 호환
  • Expo로 빠른 개발 사이클

단점

  • 캐릭터 "인터랙티브" 애니메이션은 어려움 (Lottie는 재생만)
  • 게임 엔진이 아니라 이펙트 구현에 한계
  • 복잡한 파티클/이펙트는 Skia로 직접 코딩 필요
  • 말씀하신 리액션 시스템 구현 난이도 높음

🎯 추천 변경: 게임성 중심이면 A(Unity) 또는 B(Flutter+Flame)

캐릭터가 감정표현을 하고, 실시간으로 약올리고, 오답에 리액션하는 것은 게임 엔진의 영역입니다.

Unity(A)는 가장 강력하지만 C# 학습 + 무거운 빌드가 트레이드오프.
Flutter+Flame(B)은 가볍고 빠르면서도 Rive로 듀오링고급 캐릭터 애니메이션 가능.
React Native(C)는 개발은 쉽지만 말씀하신 게임성 구현에 한계가 있습니다.
